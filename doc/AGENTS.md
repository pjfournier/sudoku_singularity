# Sudoku Singularity - Agent Instructions

Read:
- /docs/SUDOKU_SINGULARITY_ARCHITECTURE.md

Primary development goals:
- Preserve the Synergism engine architecture where practical
- Build an original atmospheric Sudoku-themed incremental game
- Prioritize gameplay feel over large-scale refactors
- Keep all changes incremental and reviewable
- Avoid touching late-game systems initially
- Preserve build stability

Current priority:
Build a playable vertical slice including:
- Number Points (NP)
- Sudoku grid visualization
- Solver upgrades
- Board Completion prestige
- Solved Boards
- Solver Logs
- Auto Board Submission

Do not implement yet:
- Variant Sudoku
- Observer Nodes
- Ghost Processes
- Impossible States
- Full singularity systems
- Procedural puzzle generation
- Large architecture rewrites

Tone:
- analytical
- atmospheric
- slightly humorous
- increasingly recursive
- subtle machine introspection

Avoid:
- horror clichés
- generic cyberpunk
- direct Cracking the Cryptic imitation
- excessive UI complexity
